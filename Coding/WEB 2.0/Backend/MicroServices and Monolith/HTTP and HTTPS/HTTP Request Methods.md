# What ?
- It indicates the purpose of the request and what is expected if the request is successful.
- Most common methods are `GET` and `POST`, for retrieving and sending data to the servers.

---
# HTTP Methods
#### **GET**
- Used to **retrieve data** from the server.
- It’s _read-only_ — should not change data on the server.
- Example: Fetch all tweets, load a YouTube video, read comments.
#### **POST**
- Used to **send data** to the server to create a new resource.
- Example: Create a tweet, upload a video, submit a comment.
#### **PATCH**
- Used to **partially update** an existing resource.
- Example: Update only the bio field of a user profile.
#### **PUT**
- Used to **completely replace** an existing resource (or create it if it doesn’t exist).
- Example: Replace an entire tweet object with new data.
#### **DELETE**
- Used to **remove data** from the server.
- Example: Delete a tweet, remove a video, delete a comment.
