# SQL v/s RDBMS

- **SQL** = language (used to query/manipulate data)
- **RDBMS** = software system (stores, manages, and processes the data)
- SQL is the _interface_; RDBMS is the _engine_

**Analogy:**
- RDBMS = kitchen
- SQL = the order you give to the kitchen

> **SQL** is the language used to interact with **RDBMS**.

**Key distinctions:**

| SQL        | RDBMS                  |                              |
| ---------- | ---------------------- | ---------------------------- |
| Type       | Language               | Software                     |
| Role       | Query/manipulate data  | Store/manage data            |
| Examples   | SELECT, INSERT, DDL... | MySQL, PostgreSQL, Oracle    |
| Dependency | Needs an RDBMS to run  | Can exist without SQL (rare) |
