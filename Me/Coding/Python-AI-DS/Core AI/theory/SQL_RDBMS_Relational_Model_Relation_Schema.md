# Relation Schema

- Defines the **design and structure** of a relation
- Contains: relation name + all its attributes
- Example: `Customer(cust_id, name, email, contact_no)`
- Schema = blueprint of the table — no actual data, just structure

**Schema ≠ Instance:**
- Schema = the structure (fixed)
- Instance = the actual rows at a point in time (changes constantly)
