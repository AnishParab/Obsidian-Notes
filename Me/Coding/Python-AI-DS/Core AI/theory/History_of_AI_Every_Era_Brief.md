# Eras of AI

| Era         | Paradigm          | Mechanism               | Limitation                     |
| ----------- | ----------------- | ----------------------- | ------------------------------ |
| 1950s–60s   | Symbolic AI       | Rules, logic, search    | Combinatorial explosion        |
| 1960s–70s   | Perceptrons       | Linear models           | Cannot model non-linearity     |
| 1970s–80s   | Expert Systems    | Rule databases          | Knowledge bottleneck           |
| 1990s–2000s | Statistical ML    | Probabilistic models    | Feature engineering dependency |
| 2010s–      | Deep Learning     | Representation learning | Data + compute cost            |
| 2020s–      | Foundation Models | Pretraining at scale    | Alignment, control, cost       |

---
# Before 1950

- Only **pattern recognition** existed.
- No actual intelligence.

---
# 1950 — Intelligence as Behavior

- Alan Turing
- **Turing Test**: intelligence defined by indistinguishable responses
- Limitation: tests imitation, not reasoning

---
# 1956 — Field Formation

- Dartmouth Conference
- AI formally established
- Dominant idea: **symbolic reasoning**

---
# Late 1950s — Lisp

- Lisp
- Enables symbolic computation via recursion and lists
- Matches tree-based logic structures

---
# 1960s — Early NLP Illusion

- ELIZA
- Pattern substitution system
- No semantic understanding

---
# 1960s–70s — Perceptron Limits

- Linear classifiers only
- Failure: cannot represent XOR
- Triggered reduced funding (AI winter)

---
# 1970s–80s — **Expert Systems**

- Prolog
- Knowledge encoded as rules
- Failure:
    - Manual rule creation does not scale
    - Fragile outside domain

---
# 1997 — Search-Based Intelligence

- Deep Blue defeats Garry Kasparov
- Approach:
    - Brute-force search + heuristics
- Not learning-based

---
# 1990s–2000s — Statistical ML

- Shift: logic → probability
- Models:
    - SVM, decision trees, Bayesian methods
- Constraint:
    - Performance limited by feature design

---
# 2011 — Probabilistic NLP

- IBM Watson wins Jeopardy!
- Pipeline-based statistical reasoning
- Not end-to-end learning

---
# 2010s — Deep Learning

- Core idea:
    - Learn hierarchical representations automatically
- Enablers:
    - GPUs, large datasets, backpropagation
- Limitations:
    - Data inefficiency
    - Poor robustness

---
# 2020s — Foundation Models / GenAI

- Pretraining on large corpora → general-purpose models
- Capabilities:
    - Text, image, audio generation
- Failure modes:
    - Hallucination
    - Alignment issues
    - High compute cost
