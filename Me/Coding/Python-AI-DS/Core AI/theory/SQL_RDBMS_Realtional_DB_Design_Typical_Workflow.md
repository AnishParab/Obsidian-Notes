# Relational DB Design Typical Workflow

- **Step 1 — ER Model:** Design your entities, attributes, and relationships as an ER diagram
- **Step 2 — Relational Model:** Convert the ER model into tables (relations)
- **Step 3 — RDBMS:** Software implementation of the relational model — MySQL, PostgreSQL, etc.

**Flow:**
```
ER Model → Relational Model → RDBMS
```

- ER Model = blueprint
- Relational Model = logical structure derived from blueprint
- RDBMS = the actual running system
