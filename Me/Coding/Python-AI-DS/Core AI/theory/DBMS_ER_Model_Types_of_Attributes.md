# Types of Attributes

![[er_types_of_attributes.excalidraw|700]]

---
# Simple

> Attributes which can't be divided further.

- Example: Customer's account number in a bank, Student's Roll number etc.

---
# Composite

> Can be divided into subparts (that is other attributes)

- Example: Name of a person, can be divided into first-name, middle-name, last-name.
- If user wants to refer to an entire attribute or to only a component of the attribute.
- Address can be divided into street, city, state, PIN code.

---
# Single-valued

> Only one value attribute.

- Example: Student ID, loan-number for a loan.

---
# Multi-valued

> Attribute having more than one value.

- Example: phone-number, nominee-name on some insurance, dependent-name etc.
- Limit constraint may be applied, upper or lower limits.

---
# Derived

> Value of this type of attribute can be derived from the value of other related attributes.

- Example: Age, loan-age, membership-period etc.

---
# NULL Value

> An attribute takes a null value when an entity does not have a value for it.

> It may indicate **not applicable**, **value doesn't exist**. 
Example: person having no middle-name.

> It may indicate **unknown**.
> 1. Unknown can indicate missing entry, example: name value of a customer is NULL, means it is missing as name must have some value.
> 2. Not known, salary attribute value of an employee is null, means it is not known yet.
