# `CLAUDE.md` File: Good Practices

> Treat it like a living document. Build it organically, not upfront. Keep making changes as the project updates.

> Correct once, then codify. i.e. if Claude makes a mistake then tell it to change the `CLAUDE.md` file so that it doesn't repeat it's mistakes again.

> Audit periodically (weekly or monthly)

- File creation
[[Applied_AI_Claude_Code_CLAUDE_md_File_Creation]]

- Commit it to `git`

- Only put *universally applicable* things to it
[[Applied_AI_Claude_Code_CLAUDE_md_File_Content_and_Structure]]

- Use of **IMPORTANT** tag
Use emphasis *sparingly* for criticla rules.
[[Applied_AI_Claude_Code_CLAUDE_md_File_important_tag]]

- Keep the file **short** `under ~200 lines`
As instruction count increases, instruction-following quality decreases uniformly.
