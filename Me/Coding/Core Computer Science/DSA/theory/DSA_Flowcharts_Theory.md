# DSA_Flowchart_Theory

- Visual representation of an algorithm or process
- Uses standardized symbols connected by arrows (flow of control)

### Components
- Terminal (Oval)      → Start / End
- Process (Rectangle) → Computation or assignment (x ← x + 1)
- Decision (Diamond)  → Condition — Yes/No or True/False branch
- I/O (Parallelogram) → Input / Output operations
- Arrow (→)           → Flow direction between steps

### Rules
- Every flowchart has exactly one Start and one End
- Decision block always has two exits (Yes/No)
- Flow goes top → bottom by default; arrows indicate otherwise
