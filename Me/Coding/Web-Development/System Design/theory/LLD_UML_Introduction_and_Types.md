# UML

- **UML** = *Unified Modeling Language* — diagrammatically express an application's structure and behaviour
- Two categories: Structural (static) and Behavioural (dynamic)

---
# **Structural Diagrams (Static)**

- Class Diagram
- Object Diagram
- Component Diagram
- Composite Structure Diagram
- Deployment Diagram
- Package Diagram
- Profile Diagram

---
# **Behavioural Diagrams (Dynamic)**

- Sequence Diagram
- Use Case Diagram
- Activity Diagram
- State Machine Diagram
- Communication Diagram
- Interaction Overview Diagram
- Timing Diagram

---
### Most used in LLD

- Class Diagram — structure of classes, relationships, attributes, methods
- Sequence Diagram — flow of messages between objects over time
