# History of Programming

- History of programming = evolution of how humans talk to machines
- Each paradigm solved the limitations of the previous one

**Machine Code**
- Raw binary (0s and 1s)
- Tedious, error-prone, not scalable
- Directly executed by CPU — no abstraction

**Assembly Language**
- Replaced binary with mnemonics (e.g. `MOV`, `ADD`)
- Still tightly coupled to hardware architecture
- Not portable — write once, runs on one machine type
- Missing high-level constructs (loops, conditionals as we know them)

**Procedural Programming**
- Code = ordered set of instructions (top-down flow)
- Introduced: functions, loops, conditionals, variables, scope
- Still no encapsulation — data and logic are separate
- Example: C

**Object-Oriented Programming**
- Modelled code around real-world entities (objects)
- Introduced: encapsulation, inheritance, polymorphism, abstraction
- Data security via access modifiers (private/public/protected)
- Highly reusable and scalable — class hierarchies, interfaces
- Example: Java, C++, Python
